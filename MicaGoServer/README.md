# MicaGoServer

A lightweight Go-based **iMessage relay server**. It runs on a Mac with access
to the local Messages database and exposes a small, stable, **Mica-native**
HTTP + WebSocket API. The current first-party client is the Flutter Android app.

The data flow is one-directional and simple:

```
chat.db  ->  sync loop  ->  relay.db  ->  REST API / WebSocket  ->  clients
```

## What MicaGoServer intentionally avoids

MicaGoServer is deliberately conservative and **not** a BlueBubbles clone. It
does **not** include:

- BlueBubbles client/server API compatibility (route shapes, envelopes, event names)
- Socket.IO (the realtime layer is a plain WebSocket)
- Any hosted account or cloud relay — connections are LAN or your own tunnel
- Electron / web admin UI
- Typing indicators, sending reactions, or group management (private‑API surface
  we chose not to take on)

Some capabilities are **optional, off by default, and user‑owned** (never cloud
bootstrapped):

- **Firebase/FCM push** — your own Firebase project; a thin wake signal, with
  message content still arriving over the socket / delta sync.
- **Edit / Unsend / Delete** — a small bundled IMCore helper (private API),
  capability‑gated; hidden when your Mac doesn't grant access.

See [`docs/micago-feature-decision-matrix.md`](docs/micago-feature-decision-matrix.md)
for the rationale.

## Running locally

Requires Go 1.24+ and macOS with Full Disk Access granted to the terminal (so
the process can read `~/Library/Messages/chat.db`).

```bash
cd micago-server
go run ./cmd/micago          # starts on 0.0.0.0:3000 (LAN-reachable) by default
```

Useful flags (all optional; config file values are used otherwise):

| Flag | Purpose |
| --- | --- |
| `--addr` | HTTP listen address (default `0.0.0.0:3000`, LAN-reachable) |
| `--token` | Override the bearer token |
| `--disable-auth` | Disable auth — **localhost only** |
| `--public-url` | Public base URL reported by `/api/server/info` |
| `--sync-interval` | Override periodic sync interval |
| `--disable-sync-loop` | Disable the periodic sync loop |
| `--sync-once` | Run one relay sync and exit |

## Configuration and token

On first run the server creates `~/.micago/config.yaml` (mode `0600`) with a
randomly generated bearer token, alongside the relay database at
`~/.micago/relay.db`. Edit the YAML to change the listen address, sync interval,
notification provider, webhook URL, etc.

```yaml
server:
  addr: "0.0.0.0:3000"
auth:
  token: "<auto-generated hex token>"
```

## Authentication

A single shared **bearer token** guards every endpoint except
`GET /api/health`.

- REST: send `Authorization: Bearer <token>`.
- WebSocket (`/ws`): send the same `Authorization` header, **or** a
  `?token=<token>` query parameter (for browser clients that cannot set
  WebSocket headers).

Auth may be disabled for localhost-only development with `--disable-auth`.
Details: [`docs/spec-v0.6.0-security.md`](docs/spec-v0.6.0-security.md).

## Smoke scripts

Each script reads the token from `~/.micago/config.yaml` and targets
`http://127.0.0.1:3000` (override with `BASE_URL`). Start the server first, then
from the `micago-server` directory:

```bash
# v0.6 — security / auth behavior
zsh scripts/smoke-v0.6-security.sh

# v0.7 — device registry (register / list / heartbeat / patch / delete)
zsh scripts/smoke-v0.7-devices.sh

# v0.8 — notification provider (test-push; webhook if configured)
zsh scripts/smoke-v0.8-notify.sh
```

## macOS companion app

The Mac-side control surface is a native SwiftUI app under
[`../micago-mac-companion/`](../micago-mac-companion/) — **not** a WebUI. It
launches/stops this server binary and reads its local control API
(`/api/server/status`, `/api/server/urls`, `/api/devices`, …) to show status,
**connection endpoints** (local/LAN/optional public), the bearer token,
registered devices, notification provider status, and permission diagnostics.
See [`docs/spec-v0.10.0-mac-companion.md`](docs/spec-v0.10.0-mac-companion.md)
and [`docs/spec-v0.11.0-connection-endpoints.md`](docs/spec-v0.11.0-connection-endpoints.md).
