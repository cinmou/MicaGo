# MicaGo 快速上手

[English](getting-started.md) · **简体中文** · [繁體中文](getting-started.zh-Hant.md)

本指南将带你完成第一次 MicaGo 配置，以及测试每条连接的推荐顺序。

## MicaGo（现阶段）能做什么

MicaGo 在你的 Mac 上运行一个小型服务器，通过一条私密、受令牌保护的连接，把你的
「信息」数据暴露给你 **自己的** 设备。Mac 上的一个伴侣应用负责启动/停止服务器，
并告诉你如何连接。

你可以通过三种方式访问这台 Mac：

1. **本机 Mac（本地）** —— 在 Mac 本机上访问。
2. **局域网（同一 Wi‑Fi）** —— 从同一网络中的另一台设备访问。
3. **公网（远程）** —— 借助你自行搭建的域名与隧道，从任何地方访问（参见
   [使用 Cloudflare Tunnel 远程访问](remote-access-cloudflare.md)）。

没有 MicaGo 云。你的数据只在你的 Mac 与你的设备之间流转。

## 你需要准备

- 一台 **已登录「信息」的 Mac**（「信息」应用中的 iMessage 可正常工作）。
- 已安装并运行的 **MicaGo Mac 应用**（伴侣/服务器）。
- Mac 应用所需的 **macOS 权限**。如果缺少某项权限，Mac 应用会提示你并链接到对应的
  「设置」面板。一般来说，应用需要读取「信息」数据库的权限（完全磁盘访问权限），
  以及控制「信息」以进行发送的权限。授予应用所请求的权限，然后重新启动服务器。
- 一台用于测试移动客户端的 **安卓设备**（可选，但推荐）。

## Mac 应用逐卡片说明

打开 MicaGo Mac 应用。**服务器** 与 **连接** 界面被组织为若干卡片：

- **服务器运行时（Server Runtime）** —— 启动 / 停止 / 重启、运行状态、服务器正在
  **监听** 的地址，以及（折叠的）近期日志输出。后端可执行文件位于 **高级** 折叠项
  之下 —— 你通常无需触碰它。
- **服务器绑定地址（Server Bind Address）** —— 选择谁可以访问服务器：
  - **仅本机 Mac**（`127.0.0.1:<端口>`）—— 仅此 Mac 上的应用。
  - **本地网络**（`0.0.0.0:<端口>`）—— 同一 Wi‑Fi 上的设备（会增加局域网端点）。
  - **自定义** —— 进阶的主机/端口。
  更改后会显示 **需要重启**；请使用 **保存并重启**。*（即使选择「仅本机 Mac」，
  通过 Cloudflare Tunnel 的远程访问仍然可用，因为 cloudflared 运行在这台 Mac 上。）*
- **连接端点（Connection Endpoints）** —— 三个区块：**本地 / 回环**、
  **局域网 / 同一 Wi‑Fi**、以及 **公网 / 远程**，每个都带复制按钮。公网区块是你
  粘贴并校验远程地址的地方。
- **客户端设置（Pair Android）** —— 选择一个端点（自动 / 本地 / 局域网 / 公网），
  然后 **显示二维码** 或 **复制设置 JSON** 来配对手机。**bearer 令牌** 以掩码形式
  显示，并带有「显示/复制」操作。

> ⚠️ **请保护好你的令牌。** bearer 令牌实际上就是一个密码。它默认以掩码显示 ——
> 只在无人窥视时才显示它，并且绝不要把它贴进截图、日志、缺陷报告或聊天中。如果
> 泄露，请生成一个新令牌并重新配对你的设备。

## 首次连接的几种方式

- **本机 Mac / 本地** —— `http://127.0.0.1:<端口>`。确认服务器存活的最快方式。
  注意：`127.0.0.1` 只在 **Mac 本机** 上有效，无法从你的手机访问。
- **局域网 / 同一 Wi‑Fi** —— `http://<Mac局域网IP>:<端口>`。从同一网络上的手机或
  笔记本电脑使用。可在 Mac 应用的连接列表，或 macOS「系统设置」→「网络」中找到
  `<Mac局域网IP>`。
- **公网 / 远程域名** —— `https://micago.example.com`。在你完成 Cloudflare 指南后，
  从移动数据或任意外部网络使用。

## 推荐路径

按以下顺序测试连接 —— 每一步都建立在前一步之上：

1. **测试本地服务器。** 在 Mac 上，确认服务器正在运行且可在
   `http://127.0.0.1:<端口>` 访问。
2. **测试局域网。** 从同一 Wi‑Fi 上的另一台设备访问
   `http://<Mac局域网IP>:<端口>`，并确认令牌被接受。
3. **设置远程地址**（可选）。按
   [使用 Cloudflare Tunnel 远程访问](remote-access-cloudflare.md) 获得
   `https://micago.example.com`，然后在 **连接端点 → 公网 / 远程** 粘贴该地址、
   **保存**，并点击 **校验公网地址**。当它显示 **可达** 时，公网即可用于配对。
4. **配对安卓客户端。** 在 **客户端设置** 中选择端点（**自动** 会在公网可达时优先
   选择公网，否则局域网，再否则本地），然后 **显示二维码** 并从安卓应用扫描 ——
   或 **复制设置 JSON**。参见 [安卓客户端连接](android-client-connection.md)。
5. **验证 WebSocket。** 在安卓应用中，确认实时连接显示 **已连接**，且调试面板列出
   收到的事件。

如需以上全部内容的精确、可复制粘贴的清单，参见
[手动测试流程](manual-test-flow.md)。

## 成功的样子

- 服务器的健康检查无需令牌即可响应。
- 你的令牌被认证检查接受。
- 安卓应用的 WebSocket 显示 **已连接**，并在调试面板中记录事件。

如果某处不工作，每份指南都有「故障排查」章节，而
[手动测试流程](manual-test-flow.md) 能帮你定位是哪一步失败了。
