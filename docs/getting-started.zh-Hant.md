# MicaGo 快速上手

[English](getting-started.md) · [简体中文](getting-started.zh-Hans.md) · **繁體中文**

本指南會帶你完成第一次 MicaGo 設定，以及測試每條連線的建議順序。

## MicaGo（現階段）能做什麼

MicaGo 在你的 Mac 上執行一個小型伺服器，透過一條私密、受權杖保護的連線，把你的
「訊息」資料開放給你 **自己的** 裝置。Mac 上的一個伴隨程式負責啟動/停止伺服器，
並告訴你如何連線。

你可以透過三種方式存取這台 Mac：

1. **本機 Mac（本機）** —— 在 Mac 本機上存取。
2. **區域網路（同一 Wi‑Fi）** —— 從同一網路中的另一台裝置存取。
3. **公開網路（遠端）** —— 借助你自行架設的網域與通道，從任何地方存取（參見
   [使用 Cloudflare Tunnel 遠端存取](remote-access-cloudflare.md)）。

沒有 MicaGo 雲端。你的資料只在你的 Mac 與你的裝置之間流轉。

## 你需要準備

- 一台 **已登入「訊息」的 Mac**（「訊息」應用程式中的 iMessage 可正常運作）。
- 已安裝並執行的 **MicaGo Mac 程式**（伴隨/伺服器）。
- Mac 程式所需的 **macOS 權限**。如果缺少某項權限，Mac 程式會提示你並連結到對應的
  「設定」面板。一般而言，程式需要讀取「訊息」資料庫的權限（完全取用磁碟權限），
  以及控制「訊息」以進行傳送的權限。授予程式所要求的權限，然後重新啟動伺服器。
- 一台用於測試行動用戶端的 **Android 裝置**（選用，但建議）。

## Mac 程式逐卡片說明

開啟 MicaGo Mac 程式。**伺服器** 與 **連線** 畫面被組織為數個卡片：

- **伺服器執行階段（Server Runtime）** —— 啟動 / 停止 / 重新啟動、執行狀態、伺服器
  正在 **監聽** 的位址，以及（摺疊的）近期日誌輸出。後端執行檔位於 **進階** 摺疊項
  之下 —— 你通常無需碰它。
- **伺服器繫結位址（Server Bind Address）** —— 選擇誰可以存取伺服器：
  - **僅本機 Mac**（`127.0.0.1:<連接埠>`）—— 僅此 Mac 上的程式。
  - **本機網路**（`0.0.0.0:<連接埠>`）—— 同一 Wi‑Fi 上的裝置（會增加區域網路端點）。
  - **自訂** —— 進階的主機/連接埠。
  變更後會顯示 **需要重新啟動**；請使用 **儲存並重新啟動**。*（即使選擇「僅本機
  Mac」，透過 Cloudflare Tunnel 的遠端存取仍然可用，因為 cloudflared 執行在這台
  Mac 上。）*
- **連線端點（Connection Endpoints）** —— 三個區塊：**本機 / 回送**、
  **區域網路 / 同一 Wi‑Fi**、以及 **公開網路 / 遠端**，每個都帶複製按鈕。公開網路
  區塊是你貼上並驗證遠端網址的地方。
- **用戶端設定（Pair Android）** —— 選擇一個端點（自動 / 本機 / 區域網路 / 公開），
  然後 **顯示 QR code** 或 **複製設定 JSON** 來配對手機。**bearer 權杖** 以遮罩形式
  顯示，並帶有「顯示/複製」操作。

> ⚠️ **請保護好你的權杖。** bearer 權杖實際上就是一組密碼。它預設以遮罩顯示 ——
> 只在無人窺視時才顯示它，並且絕不要把它貼進截圖、日誌、錯誤回報或聊天中。如果
> 外洩，請產生一組新權杖並重新配對你的裝置。

## 首次連線的幾種方式

- **本機 Mac / 本機** —— `http://127.0.0.1:<連接埠>`。確認伺服器存活的最快方式。
  注意：`127.0.0.1` 只在 **Mac 本機** 上有效，無法從你的手機存取。
- **區域網路 / 同一 Wi‑Fi** —— `http://<Mac區域網路IP>:<連接埠>`。從同一網路上的
  手機或筆記型電腦使用。可在 Mac 程式的連線清單，或 macOS「系統設定」→「網路」中
  找到 `<Mac區域網路IP>`。
- **公開網路 / 遠端網域** —— `https://micago.example.com`。在你完成 Cloudflare 指南
  後，從行動數據或任意外部網路使用。

## 建議路徑

依以下順序測試連線 —— 每一步都建立在前一步之上：

1. **測試本機伺服器。** 在 Mac 上，確認伺服器正在執行且可在
   `http://127.0.0.1:<連接埠>` 存取。
2. **測試區域網路。** 從同一 Wi‑Fi 上的另一台裝置存取
   `http://<Mac區域網路IP>:<連接埠>`，並確認權杖被接受。
3. **設定遠端網址**（選用）。依
   [使用 Cloudflare Tunnel 遠端存取](remote-access-cloudflare.md) 取得
   `https://micago.example.com`，然後在 **連線端點 → 公開網路 / 遠端** 貼上該網址、
   **儲存**，並點擊 **驗證公開網址**。當它顯示 **可達** 時，公開網路即可用於配對。
4. **配對 Android 用戶端。** 在 **用戶端設定** 中選擇端點（**自動** 會在公開網路可達
   時優先選擇公開網路，否則區域網路，再否則本機），然後 **顯示 QR code** 並從
   Android 程式掃描 —— 或 **複製設定 JSON**。參見
   [Android 用戶端連線](android-client-connection.md)。
5. **驗證 WebSocket。** 在 Android 程式中，確認即時連線顯示 **已連線**，且除錯面板
   列出收到的事件。

如需以上全部內容的精確、可複製貼上的清單，參見
[手動測試流程](manual-test-flow.md)。

## 成功的樣子

- 伺服器的健康檢查無需權杖即可回應。
- 你的權杖被認證檢查接受。
- Android 程式的 WebSocket 顯示 **已連線**，並在除錯面板中記錄事件。

如果某處無法運作，每份指南都有「疑難排解」章節，而
[手動測試流程](manual-test-flow.md) 能幫你找出是哪一步失敗了。
